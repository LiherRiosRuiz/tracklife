### 5. Fix quevedo.mjs — enlaces rotos por code spans

**Problema**: Quevedo mostraba 2 enlaces rotos (`[[Nota\|Alias]]`).

**Causa**: `buildVaultGraph` parseaba wikilinks sobre el contenido raw,
incluyendo ejemplos de sintaxis dentro de code spans en `docs/Cronica.md`
y `docs/Quevedo.md`. El string `[[Nota\|Alias]]` escrito como ejemplo
se trataba como enlace real → destino `Nota\` no existia.

**Fix** en `.sdd/cli/quevedo.mjs`, funcion `buildVaultGraph`:
strip de bloques de codigo e inline code spans antes del regex de wikilinks:
```javascript
const content = raw
  .replace(/```[\s\S]*?```/g, "")
  .replace(/`[^`\n]+`/g, "");
```

**Resultado**: 0 rotos, 0 huerfanas en el vault.

**Fix de logout (frontend):** Ya estaba aplicado en sesion anterior. `lib/auth.tsx`
llama a `api.logout(token).catch(() => {})` antes de limpiar localStorage.

**Decision tecnica notable:** El test de revocacion de token requirio
`$this->app['auth']->forgetGuards()` entre el logout y la segunda peticion.
El auth guard de Laravel cachea el usuario en memoria durante el ciclo de vida
del proceso. En produccion cada HTTP request es un proceso separado —
`forgetGuards()` simula ese boundary en el entorno de tests. El token si se
elimina de MongoDB (verificado con `PersonalAccessToken::count() == 0`).
