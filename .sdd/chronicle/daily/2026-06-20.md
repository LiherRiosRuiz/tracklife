## Sesion: Indices MongoDB + Refactor DashboardController (Paso 3)

**Objetivo**: Tercer paso del plan de mejora de TrackLife — infra de base de datos y limpieza arquitectonica.

**Contexto**:
- Ninguna coleccion de MongoDB tiene indices declarados. Todas las queries filtran por `user_id` como minimo — sin indice es un full collection scan en cada request.
- `DashboardController::index()` llama a `app(MacroController::class)->dailyProgress($request)` — un controller invocando a otro controller directamente, violacion del patron del workspace.
- Con 5 tests de Feature en verde (AuthTest, ExerciseTest, WorkoutPlanTest, MealTest, MacroTest), el refactor puede hacerse con confianza.

**Tareas de esta sesion**:
1. Crear migracion de indices MongoDB para todas las colecciones principales
2. Extraer `MacroService::dailyProgress()` del MacroController
3. Refactorizar DashboardController para usar el nuevo MacroService
4. Verificar que `php artisan test tests/Feature` sigue en verde

**Stack**: Laravel 13 · MongoDB · php artisan migrate
